================================================================================
AWS → OCI TRANSLATION NOTES
================================================================================

Key differences documented in index.md:

1. POLICY LANGUAGE:
   AWS: JSON with Effect/Action/Resource/Condition
   OCI: Natural language (Allow group X to manage Y in compartment Z)

2. PERMISSION MODEL:
   AWS: Fine-grained actions (s3:GetObject, ec2:DescribeInstances)
   OCI: Broad verbs (inspect/read/use/manage) + resource-types

3. RESOURCE SCOPING:
   AWS: Account + ARN hierarchy
   OCI: Compartment hierarchy

4. POLICY ATTACHMENT:
   AWS: Attach to users/groups/roles or resources
   OCI: Attach to compartments, apply to groups

5. CROSS-ACCOUNT:
   AWS: AssumeRole with trust policies
   OCI: Endorse (source) + Admit (destination)
